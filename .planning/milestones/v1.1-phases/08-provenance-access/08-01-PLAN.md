**GREEN:** Implement in query_cli.go:

1. Add `MentionDetail` struct:
```go
type MentionDetail struct {
    FilePath        string  `json:"file_path"`
    DetectionMethod string  `json:"detection_method"`
    Confidence      float64 `json:"confidence"`
    Context         string  `json:"context,omitempty"`
}
```

2. Extend `ImpactNode` struct with two new fields (both omitempty):
```go
Mentions     []MentionDetail `json:"mentions,omitempty"`
MentionCount int             `json:"mention_count,omitempty"`
```

3. Add `decorateWithMentions(nodes []ImpactNode, mentions map[string][]ComponentMention, limit int)` function:
   - For each node, look up `mentions[node.Name]`
   - Set `MentionCount` to total count (before truncation)
   - Apply limit (if limit > 0 and len > limit, truncate)
   - Map `ComponentMention` to `MentionDetail`: `file_path` from `FilePath`, `detection_method` from `DetectedBy`, `confidence` from `Confidence`, `context` from `HeadingHierarchy`
   - Assign mapped slice to `node.Mentions`

4. In `cmdQueryImpact`:
   - Add flags: `--include-provenance` (bool, default false), `--max-mentions` (int, default 5)
   - After BFS traversal produces nodes, if `--include-provenance` is set:
     - Open the DB for the resolved graph (use the same DB path resolution as LoadStoredGraph — the graph directory contains graph.db)
     - Call `db.LoadComponentMentions()`
     - Call `decorateWithMentions(nodes, mentions, maxMentions)`
     - Close the DB
   - If loading mentions fails, print warning to stderr and continue without mentions (non-fatal)

5. In `cmdQueryDependencies`:
   - Same flag additions and decoration logic as cmdQueryImpact

IMPORTANT: Do NOT modify BFS traversal functions. Decoration happens after BFS returns.

IMPORTANT: The `component_mentions` table uses `detected_by` column but output JSON must use `detection_method` field name (per user decision). The MentionDetail struct handles this mapping.

IMPORTANT: `MentionCount` is `int` with `omitempty` — zero value suppresses it when provenance is not requested. This is correct because without `--include-provenance`, MentionCount stays 0 and is omitted.
