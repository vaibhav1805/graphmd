## Cross-Phase Integration Issues

### Orphaned Exports (Tech Debt)

1. **`ExecuteImpact` / `ExecuteCrawl`** (Phase 2, `query.go`)
   - Exported, tested (10+ unit tests), never called by any CLI command
   - Phase 4 re-implemented traversal with correct semantics (reverse BFS for "what breaks")
   - Phase 5 uses `ComputeCrawlStats` not `ExecuteCrawl`
   - **Impact:** Dead code. No functional gap, but maintenance risk.

2. **`QueryResult`, `AffectedNode`, `QueryEdge`** (Phase 2, `types.go`)
   - Defined and tested, never used by Phase 4 CLI output
   - Phase 4 defines its own `ImpactResult`, `ImpactNode`, `EnrichedRelationship`

### Wiring Gaps (Non-Critical)

3. **Cycle reporting absent from CLI** (REL-03)
   - Phase 2's `TraverseDFS` marks cyclic edges and records cycles
   - Phase 4's BFS prevents infinite loops but doesn't report cycles to users
   - Cycles are silently dropped, not flagged

4. **Dangling edge drop at export** (Phase 3)
   - `SaveGraph` silently drops edges where endpoint node doesn't exist
   - No warning to operator about lost relationships

5. **component_mentions not queryable** (REL-02)
   - Edge-level provenance (source_file, extraction_method) works in query results
   - Component-level mention provenance stored in DB but inaccessible via CLI
