  <behavior>
    The PythonParser implements the LanguageParser interface (Name/Extensions/ParseFile).

    Import tracking (two-phase scan):
    - `import requests` -> importMap["requests"] = "requests"
    - `import requests as req` -> importMap["req"] = "requests"
    - `from kafka import KafkaProducer` -> importMap["KafkaProducer"] = "kafka"
    - `from kafka import KafkaProducer as KP` -> importMap["KP"] = "kafka.KafkaProducer"
    - `from redis import Redis` -> importMap["Redis"] = "redis.Redis"

    Detection patterns (declarative, following GoParser template):
    - HTTP: requests.get/post/put/delete/patch (0.9), httpx.get/post/put/delete (0.9)
    - DB: psycopg2.connect (0.85), asyncpg.connect/create_pool (0.85), pymongo.MongoClient (0.85), sqlalchemy.create_engine (0.85), pymysql.connect (0.85)
    - Cache: redis.Redis/StrictRedis/from_url (0.85), pymemcache Client (0.85)
    - Queue: kafka.KafkaProducer/KafkaConsumer (0.85), pika.BlockingConnection (0.85), boto3.client with "sqs" arg (0.85)

    Comment hints: `# Calls payment-api`, `# Depends on auth-service` -> confidence 0.4

    False positive control:
    - Skip lines starting with # (comments)
    - Strip inline comments before matching
    - Skip lines starting with @ (decorators)
    - Skip test files: *_test.py, test_*.py, conftest.py
    - No import-only signals (require usage evidence)

    Target extraction: Parse URL from first string argument using net/url, extract hostname.

    Test cases (inline Python source fixtures):
    - requests.get("http://payment-api:8080/pay") -> http_call, target=payment-api
    - import requests as req; req.post("http://user-svc/create") -> http_call, target=user-svc
    - psycopg2.connect("postgres://primary-db:5432/mydb") -> db_connection, target=primary-db
    - from redis import Redis; Redis(host="cache-01") -> cache_client signal
    - from kafka import KafkaProducer; KafkaProducer(bootstrap_servers="kafka-broker:9092") -> queue_producer signal
    - import only (import requests with no call) -> no signals
    - comment-only line (# requests.get("http://old")) -> no signals
    - decorator line (@app.route("/api/v1")) -> no signals
    - test file (test_handler.py) -> no signals
    - # Calls payment-api -> comment_hint signal at 0.4 confidence
  </behavior>
  <implementation>
    **patterns.go:** Define PyDetectionPattern struct (Package, Function, Kind, TargetType, Confidence, ArgIndex). Create DefaultPythonPatterns slice. Define import regex patterns (3 regexes: bare import, aliased import, from-import). Define call pattern regex and comment hint regex.

    **parser.go:** PythonParser struct with patterns map[string]PyDetectionPattern. NewPythonParser() constructor. Name() returns "python". Extensions() returns [".py"]. ParseFile():
    1. Skip test files (check filename patterns)
    2. Split content into lines
    3. First pass: build importMap from import statements using regexes
    4. Second pass: for each non-comment, non-decorator line, match package.function() calls against importMap + patterns table. Also match from-imported bare function calls (e.g., `Redis(...)` when `from redis import Redis`).
    5. Extract URL target from first string arg using net/url
    6. Scan for comment hints matching `#\s*(?:Calls|Depends on|Uses|Connects to)\s+(\S+)`
    7. Return []CodeSignal with Language="python"

    Follow GoParser's evidenceSnippet pattern for evidence field. Use compile-time interface check: `var _ code.LanguageParser = (*PythonParser)(nil)`.
  </implementation>
