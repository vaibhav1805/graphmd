Create `internal/code/connstring/` package implementing connection string parsing.

**Result type:**
```go
type Result struct {
    Host       string  // Extracted hostname or env var name
    TargetType string  // database, cache, message-broker, service, unknown
    Confidence float64 // 0.85 for parsed connection strings, 0.7 for env var refs
    Kind       string  // "conn_string" or "env_var_ref"
}
```

**Core function `Parse(raw string) (Result, bool)`:**
1. Try `net/url.Parse` first — if scheme is recognized and hostname is non-empty, map scheme to target type:
   - postgres/postgresql/mysql/mongodb/mongodb+srv → database
   - redis/rediss → cache
   - amqp/amqps/nats → message-broker
   - http/https → service
   - Unknown scheme with hostname → unknown
2. Filter out documentation/example domains: example.com, example.org, localhost, 127.0.0.1, 0.0.0.0, and common doc sites (docs.python.org, pkg.go.dev, developer.mozilla.org, etc.)
3. Fall back to DSN formats on URL parse failure:
   - MySQL DSN: `@tcp(host:port)/` → extract host, type=database
   - PostgreSQL key-value: `host=X` → extract X, type=database
   - Generic `host:port/dbname` → extract host, type=unknown
4. Fall back to host:port format (reuse the pattern from pyparser's extractURLHost)
5. Return `ok=false` if nothing matches

**Environment variable reference detection `ParseEnvVarRef(line string) []EnvVarRef`:**
- Match patterns across languages:
  - Go: `os.Getenv("VAR")`, `os.LookupEnv("VAR")`
  - Python: `os.environ["VAR"]`, `os.environ.get("VAR")`, `os.getenv("VAR")`
  - JS: `process.env.VAR`, `process.env["VAR"]`
  - Shell-style in string literals: `$VAR`, `${VAR}`
- Return struct with var name and position

**`IsConnectionEnvVar(name string) bool`:**
- Check suffixes: _URL, _DSN, _URI, _HOST, _ADDR, _ENDPOINT, _CONNECTION
- Check prefixes: DATABASE_, DB_, REDIS_, MONGO_, RABBIT_, KAFKA_, NATS_, AMQP_

Confidence: 0.85 for parsed connection strings (per CONTEXT.md), 0.7 for env var references where actual target is unknown.

**TDD approach:** Write tests FIRST covering:
- URL schemes: postgres://, redis://, mongodb://, amqp://, http://, https://

- Host:port fallback
- Filtered domains (example.com, localhost)
- Env var patterns for all three languages
- IsConnectionEnvVar positive and negative cases
- Edge cases: empty string, malformed URLs, bare hostnames

Then implement to make tests pass.
