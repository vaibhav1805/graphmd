After `analyzer.AnalyzeDir(dir)` returns all signals, add a `boostKnownComponents` function:

```go
func boostKnownComponents(signals []CodeSignal) {
    // First pass: collect all unique TargetComponent names from non-comment signals
    known := make(map[string]bool)
    for _, s := range signals {
        if s.DetectionKind != "comment_hint" {
            known[s.TargetComponent] = true
        }
    }

    // Second pass: boost comment_hint signals that reference known components
    for i := range signals {
        if signals[i].DetectionKind == "comment_hint" {
            if known[signals[i].TargetComponent] {
                signals[i].Confidence = 0.5
            }
        }
    }
}
```

Call `boostKnownComponents(signals)` in RunCodeAnalysis after the AnalyzeDir call and before the relative path adjustment loop.

This implements the two-pass approach recommended in research (Option B) — no LanguageParser interface changes needed. The boost is in-place on the signals slice.

Also add `boostKnownComponents` to integration_test.go: create a test that verifies a comment_hint signal referencing a component also found by code detection gets boosted to 0.5, while a comment_hint for an unknown component stays at 0.4.
