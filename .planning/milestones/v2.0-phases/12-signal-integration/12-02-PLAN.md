**In `export.go`, replace the Step 7b block** (currently prints summary and discards signals) with full integration:

```
if a.AnalyzeCode {
    fmt.Fprintf(os.Stderr, "  Analyzing source code...\n")
    signals, codeErr := code.RunCodeAnalysis(absFrom,
        goparser.NewGoParser(),
        pyparser.NewPythonParser(),
        jsparser.NewJSParser(),
    )
    if codeErr != nil {
        fmt.Fprintf(os.Stderr, "  Warning: code analysis failed: %v\n", codeErr)
    } else {
        code.PrintCodeSignalsSummary(os.Stderr, signals)

        // Convert code signals to DiscoveredEdge values
        sourceComponent := code.InferSourceComponent(absFrom)
        codeEdges := convertCodeSignalsToDiscovered(signals, sourceComponent)

        // Create stub nodes for targets not already in the graph
        ensureCodeTargetNodes(graph, codeEdges)

        // Merge code edges with markdown-discovered edges
        allEdges := MergeDiscoveredEdges(discovered, codeEdges)

        // Apply probabilistic OR for edges detected by both markdown and code
        applyMultiSourceConfidence(allEdges)

        // Replace the discovered slice with the merged result
        discovered = allEdges

        // Set source_type on each edge before adding to graph
        for _, de := range discovered {
            if de.Edge != nil {
                de.Edge.SourceType = determineSourceType(de)
            }
        }

        fmt.Fprintf(os.Stderr, "  Code analysis: %d signals → %d code edges, %d total merged edges\n",
            len(signals), len(codeEdges), len(discovered))

        // Save raw code signals for provenance (after DB is created in Step 8)
        // Store signals in a variable to save after DB open
    }
}
```

IMPORTANT: The code signals provenance save (`SaveCodeSignals`) must happen AFTER the database is opened in Step 8. Store the signals and sourceComponent in variables before Step 8, then call `db.SaveCodeSignals(signals, sourceComponent)` after `db.SaveGraph()` succeeds. This follows the existing pattern where the DB is opened late in the pipeline.

Also ensure that when `--analyze-code` is NOT used, edges still get `SourceType = "markdown"` set. Since `NewEdge()` now defaults SourceType to "markdown" (from Plan 01), this is automatic for markdown-discovered edges. But verify that edges added via the existing `graph.AddEdge()` path (Step 7a) work correctly.

1. Add `SourceType string` field to `EnrichedRelationship` struct:
   ```go

   ```
   If edge.SourceType is empty (pre-v6 graphs), default to "markdown".

3. Add `--source-type` flag to query subcommands (impact, dependencies, path, list):

4. Add `SourceType` parameter to `QueryEnvelopeParams` so the filter appears in query metadata:
   ```go
   SourceType string `json:"source_type,omitempty"`
   ```
