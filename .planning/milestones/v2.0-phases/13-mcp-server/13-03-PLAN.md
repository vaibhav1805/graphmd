<objective>
Fix MCP server stdin EOF race condition that prevents initialize response from being written.

Purpose: The current `StdioTransport{}` implementation causes a race when stdin is a pipe (e.g., `echo '...' | graphmd mcp`). The jsonrpc2 read goroutine sees EOF before the SDK writes the initialize response, causing the write to be silently dropped. This fix replaces `StdioTransport` with `IOTransport` using a pipe wrapper that holds stdin open until context cancellation, decoupling stdin EOF from transport EOF.

Output: Working MCP server that responds to piped initialize requests + integration test.
</objective>

In `internal/mcp/server.go`, replace the `StdioTransport` usage with an `IOTransport` that wraps stdin in a pipe. The pipe goroutine copies stdin into the pipe writer, then blocks on `<-ctx.Done()` before closing the pipe writer. This ensures the transport never sees EOF until the context is cancelled (SIGTERM/SIGINT or explicit shutdown).

Specific changes to `Run()`:

1. Add `"io"` to imports.

2. Replace the final line `return server.Run(ctx, &mcpsdk.StdioTransport{})` with:

```go
// Wrap stdin in a pipe that stays open until context cancellation.
// This prevents the SDK's jsonrpc2 layer from seeing EOF (and entering
// shuttingDown state) before it finishes writing responses.
// Without this, piped input (echo '...' | graphmd mcp) races: stdin
// EOF arrives before the initialize response is written, causing it
// to be silently dropped.
pr, pw := io.Pipe()
go func() {
    io.Copy(pw, os.Stdin)
    <-ctx.Done()
    pw.Close()
}()

return server.Run(ctx, &mcpsdk.IOTransport{
    Reader: pr,
    Writer: nopWriteCloser{origStdout},
})
```

3. Add a `nopWriteCloser` type (IOTransport requires `io.WriteCloser` but we do not want to close stdout):

```go
// nopWriteCloser wraps an io.Writer with a no-op Close method.
type nopWriteCloser struct{ w io.Writer }

func (n nopWriteCloser) Write(p []byte) (int, error) { return n.w.Write(p) }
func (n nopWriteCloser) Close() error                { return nil }
```
