<task type="auto">
  <name>Task 1: XDG storage and ZIP import pipeline</name>
  <files>internal/knowledge/xdg.go, internal/knowledge/import.go</files>
  <action>
Create `xdg.go` with:
- `GraphStorageDir() (string, error)` — returns `$XDG_DATA_HOME/graphmd/graphs/` or `~/.local/share/graphmd/graphs/` fallback. Uses `os.Getenv("XDG_DATA_HOME")`, falls back to `os.UserHomeDir() + ".local/share"`. No third-party dependencies.
- `graphStoragePath(name string) (string, error)` — returns full path for a named graph directory.
- `getCurrentGraph(storageDir string) (string, error)` — reads `current` text file to get default graph name.
- `setCurrentGraph(storageDir, name string) error` — writes `current` file with graph name.

Create `import.go` with:
- `CmdImport(args []string) error` — CLI entry point. Parses flags: positional arg for ZIP path, `--name` for graph name (default: derive from ZIP filename stem). Calls `ImportZIP`. Prints success message to stderr including graph name and component/relationship counts from metadata.
- `ImportZIP(zipPath, graphName string) error` — Core pipeline:
  1. Open ZIP with `archive/zip`, extract to temp dir
  2. Validate ZIP contains `graph.db` and `metadata.json`
  3. Parse `metadata.json` into `ExportMetadata` struct
  4. Schema validation: reject if `meta.SchemaVersion > SchemaVersion` with clear error message ("upgrade graphmd")
  5. Validate DB: open with `OpenDB`, call `LoadGraph` on a temp graph to ensure DB is readable
  6. Create destination dir via `os.MkdirAll(destDir, 0o755)` — handles first-run case
  7. Copy `graph.db` and `metadata.json` to destination (overwrite if exists, print notice to stderr: "Replacing existing graph 'name'")
  8. Update `current` marker via `setCurrentGraph`
- `LoadStoredGraph(graphName string) (*Graph, *ExportMetadata, error)` — Loads a named graph from XDG storage for query commands to use. If graphName is empty, reads `current` file. Returns structured errors for NO_GRAPH (no graph imported) and NOT_FOUND (named graph doesn't exist).
- Graph name derivation: if `--name` not provided, use `strings.TrimSuffix(filepath.Base(zipPath), filepath.Ext(zipPath))`.

Error handling:
- Invalid ZIP: `fmt.Errorf("import: open ZIP %q: %w", zipPath, err)`
- Missing files: `fmt.Errorf("import: invalid archive — missing graph.db or metadata.json")`
- Schema too new: `fmt.Errorf("import: archive schema version %d is newer than supported version %d — upgrade graphmd", meta.SchemaVersion, SchemaVersion)`
- DB validation failure: `fmt.Errorf("import: graph database validation failed: %w", err)`

Use `io.Copy` with file extraction (not `io.ReadAll`) for proper streaming of large databases.
