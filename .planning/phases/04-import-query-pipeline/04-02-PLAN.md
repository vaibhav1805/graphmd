**Response Envelope (per CONTEXT.md contract):**

```go
type QueryEnvelope struct {
    Query    QueryEnvelopeParams   `json:"query"`
    Results  interface{}           `json:"results"`
    Metadata QueryEnvelopeMetadata `json:"metadata"`
}

type QueryEnvelopeParams struct {
    Type          string  `json:"type"`
    Component     string  `json:"component,omitempty"`
    From          string  `json:"from,omitempty"`
    To            string  `json:"to,omitempty"`
    Depth         int     `json:"depth,omitempty"`
    MinConfidence float64 `json:"min_confidence,omitempty"`
}

type QueryEnvelopeMetadata struct {
    ExecutionTimeMs int64  `json:"execution_time_ms"`
    NodeCount       int    `json:"node_count"`
    EdgeCount       int    `json:"edge_count"`
    GraphName       string `json:"graph_name"`
    GraphVersion    string `json:"graph_version"`
    CreatedAt       string `json:"created_at"`
    ComponentCount  int    `json:"component_count"`
}
```

**Enriched relationship type for output (adds tier name per CONTEXT.md):**

```go
type EnrichedRelationship struct {
    From             string  `json:"from"`
    To               string  `json:"to"`
    Confidence       float64 `json:"confidence"`
    ConfidenceTier   string  `json:"confidence_tier"`
    Type             string  `json:"type"`
    SourceFile       string  `json:"source_file"`
    ExtractionMethod string  `json:"extraction_method"`
}
```

**CmdQuery(args []string) error** — Router:
- Parse first arg as subcommand: impact, dependencies, path, list
- Unknown subcommand: print usage to stderr, exit 1
- No subcommand: print usage to stderr, exit 1

**Global flags** (parsed per-subcommand via flag.NewFlagSet):
- `--graph <name>` — select stored graph (default: most recent)
- `--min-confidence <float>` — filter relationships below threshold
- `--format json|table` — output format (default: json)

**Impact subcommand** (`cmdQueryImpact`):
- Flags: `--component` (required), `--depth` (default 1, "all" maps to 100), `--min-confidence`, `--graph`, `--format`
- CRITICAL EDGE DIRECTION: The existing `ExecuteImpact` follows `BySource` (outgoing edges from root). For "if X fails, what breaks?", we need to follow `ByTarget[X]` — things that depend on X. Implement a NEW `executeImpactReverse` function that:
  1. Uses BFS/DFS following `g.ByTarget[nodeID]` (incoming edges) instead of `g.BySource`
  2. Supports depth limiting and confidence filtering (same as ExecuteImpact)
  3. Returns `*QueryResult` in the same format
  Alternatively, parameterize ExecuteImpact to accept a direction, but a new function is cleaner since we're in the CLI layer.
- Wrap result in QueryEnvelope. Enrich each relationship with `confidence_tier` from `ScoreToTier()`.
- Component not found: call `suggestComponents()` and return error JSON with suggestions.

**Dependencies subcommand** (`cmdQueryDependencies`):
- Flags: `--component` (required), `--depth` (default 1, "all" maps to 100), `--min-confidence`, `--graph`, `--format`
- Uses existing `ExecuteImpact` (which follows `BySource` — outgoing edges = what X depends on). This is semantically correct for "what does X need to work?"
- Same envelope wrapping and confidence enrichment as impact.

**Path subcommand** (`cmdQueryPath`):
- Flags: `--from` (required), `--to` (required), `--limit` (default 10), `--min-confidence`, `--graph`, `--format`
- Uses `g.FindPaths(from, to, maxDepth)` where maxDepth is a reasonable bound (e.g., 20 for path-finding).
- For each path ([]string of node IDs), look up edge between consecutive nodes to build per-hop detail:
  ```go
  type PathResult struct {
      Paths []PathInfo `json:"paths"`
      Count int        `json:"count"`
      Reason string    `json:"reason,omitempty"` // populated when no paths found
  }
  type PathInfo struct {
      Nodes           []string  `json:"nodes"`
      Hops            []HopInfo `json:"hops"`
      TotalConfidence float64   `json:"total_confidence"` // product of hop confidences
  }
  type HopInfo struct {
      From             string  `json:"from"`
      To               string  `json:"to"`
      Confidence       float64 `json:"confidence"`
      ConfidenceTier   string  `json:"confidence_tier"`
      SourceFile       string  `json:"source_file"`
      ExtractionMethod string  `json:"extraction_method"`
  }
  ```
- Edge lookup: iterate `g.BySource[from]` to find edge with `Target == to`. If not found (shouldn't happen but defensive), use zero confidence.
- Apply `--limit` to cap number of returned paths.
- Apply `--min-confidence` to filter paths where any hop is below threshold.
- **No path found:** Return success (exit 0) with `PathResult{Paths: [], Count: 0, Reason: "no path found between X and Y"}`.
- **Component not found (either from or to):** Return error JSON with suggestions for the missing component.

**List subcommand** (`cmdQueryList`):
- Flags: `--type` (optional, filter by component type), `--min-confidence` (optional), `--graph`, `--format`
- Iterate all `g.Nodes`. If `--type` is set, filter by `node.ComponentType`. If `--min-confidence` is set, only include nodes whose edges all meet threshold (or simpler: just list nodes, since confidence applies to edges not nodes — per CONTEXT.md, compose filters with AND).
- Return list of components with their type and connected edge count.
  ```go
  type ListResult struct {
      Components []ListComponent `json:"components"`
      Count      int             `json:"count"`
  }
  type ListComponent struct {
      Name          string `json:"name"`
      Type          string `json:"type"`
      IncomingEdges int    `json:"incoming_edges"`
      OutgoingEdges int    `json:"outgoing_edges"`
  }
  ```

**Fuzzy component matching** (`suggestComponents`):
- Iterate all node IDs. Score by: substring match (score 2), prefix match on first 3 chars (score 1).
- Sort by score descending, return top 5.
- Used in error responses for impact, dependencies, and path queries.

**Error formatting:**
- `writeErrorJSON(w io.Writer, message, code string, suggestions []string)` — writes `{"error": "...", "code": "...", "suggestions": [...]}` envelope.
- NO_GRAPH code when no graph imported; NOT_FOUND code when component not found.
- All errors write to stdout (JSON) and return non-nil error so the CLI exits with code 1.

**Table format** (`--format table`):
- Minimal implementation using `text/tabwriter`.
- Impact/dependencies: columns for Name, Type, Distance, Confidence, Tier.
- Path: one path per block with hop details.
- List: columns for Name, Type, Incoming, Outgoing.

**Helper: executeImpactReverse(g *Graph, root string, depth int, minConf *float64) ([]AffectedNode, []QueryEdge)**
- BFS from root following `g.ByTarget[nodeID]` (incoming edges — things that have root as a dependency).
- Track visited set for cycle safety.
- Return affected nodes with distances and filtered edges.
- This is the key function that makes "if X fails, what breaks?" work correctly.

**Test graph structure:**
```
payment-api --(depends-on)--> primary-db
payment-api --(depends-on)--> redis-cache
auth-service --(depends-on)--> primary-db
auth-service --(depends-on)--> payment-api
web-frontend --(depends-on)--> auth-service
web-frontend --(depends-on)--> payment-api
```
Edge directions: `A -> B` means A depends on B (A has outgoing edge to B).
So: if `primary-db` fails, `payment-api` and `auth-service` break (they depend on it).

Component types: payment-api=service, primary-db=database, redis-cache=cache, auth-service=service, web-frontend=service.

Set confidence scores to varied values (0.95, 0.85, 0.7, etc.) and provenance fields (source_file, extraction_method) for provenance testing.

2. `TestQueryImpact_TransitiveDependents` — Query impact on `primary-db` with `--depth all`. Should include `web-frontend` (depends on auth-service which depends on primary-db).

3. `TestQueryImpact_DifferentFromDependencies` — Critical test: query impact on `payment-api` and query dependencies on `payment-api` must return DIFFERENT results. 10. `TestQueryEnvelope_ConfidenceTier` — Verify every relationship in output has both `confidence` (numeric) and `confidence_tier` (string) fields.

12. `TestQueryError_NoGraph` — Query with no graph imported. Verify error JSON has `code: "NO_GRAPH"` and `action` field.
