  <files>internal/knowledge/crawl_cmd.go</files>
  <action>
    Create `crawl_cmd.go` in package knowledge with:

    1. **CrawlArgs struct** with fields: Input (string, --input flag), Format (string, --format flag, default "text"), FromMultiple (string, --from-multiple flag for backward compat).

    2. **ParseCrawlArgs(args []string) (*CrawlArgs, error)** using flag.NewFlagSet("crawl", flag.ContinueOnError):
       - `--input` (default ".") — source directory
       - `--format` (default "text") — output format: text or json
       - `--from-multiple` (default "") — backward compat for targeted traversal mode
       - Positional arg overrides --input (same as export pattern)

    3. **CmdCrawl(args []string) error** — the main entry point:
       - Parse args via ParseCrawlArgs
       - If FromMultiple is set, return ErrLegacyCrawl sentinel error (main.go will fall back to old behavior)
       - Resolve abs path, verify directory exists
       - **Steps 1-7 mirroring CmdExport exactly** (read export.go lines 134-218 carefully):
         - Step 1: LoadGraphmdIgnore(absInput)
         - Step 2: LoadAliasConfig(absInput)
         - Step 3: ScanDirectory(absInput, ScanConfig{IgnoreDirs, IgnoreFiles, UseDefaultIgnores: true})
         - Step 4: Build graph (NewGraph, add nodes from docs, extract edges via NewExtractor)
         - Step 5: Component detection (NewComponentDetector, DetectComponents, set node.ComponentType)
         - Step 6: Apply aliases (applyAliases if aliases exist)
         - Step 7: Run discovery (DiscoverRelationships, add edges with confidence >= 0.5 matching export default)
       - Step 8: Call ComputeCrawlStats(graph) from crawl_stats.go
       - Step 9: Format and print based on Format flag

    4. **Text formatter** — formatCrawlStatsText(stats CrawlStats, inputPath string) string:
       - Header: "Graph Summary" with input path
       - Summary line: "N components, M relationships, Quality: XX.X%"
       - Blank line
       - "Components by Type" section: for each type (sorted), show "  type (count):" then list each component ID indented with "    - id"
       - Blank line
       - "Confidence Distribution" section with ASCII bar chart:
         - For each non-zero tier: `  tier-name (low-high):  NN edges (XX.X%) ||||||||`
         - Bar chart scaled relative to max count, max width 30 chars
         - Use `|` character for bars (per CONTEXT.md example)
         - Minimum 1 char bar for non-zero counts
       - Blank line
       - "Quality Warnings" section (only if warnings exist):
         - For each warning: `  [type] message`
         - List items indented: `    - item`
       - If no warnings: "  No quality issues detected."

    5. **JSON formatter** — formatCrawlStatsJSON(stats CrawlStats, inputPath string) (string, error):
       - Use CrawlStatsJSON struct with json tags:
         ```
         {
           "summary": {
             "component_count": N,
             "relationship_count": N,
             "quality_score": XX.X,
             "input_path": "..."
           },
           "components": {
             "by_type": { "service": ["a", "b"], "database": ["c"] }
           },
           "confidence": {
             "tiers": [
               {"tier": "strong-inference", "range": [0.75, 0.95], "count": 23, "percentage": 46.0}
             ]
           },
           "quality_warnings": [
             {"type": "orphan_node", "message": "...", "count": N, "items": ["..."]}
           ]
         }
         ```
       - Use json.MarshalIndent with 2-space indent
       - Include score ranges in each tier entry (self-documenting for scripts, per user decision)

    6. **ErrLegacyCrawl** — exported sentinel error var: `var ErrLegacyCrawl = fmt.Errorf("legacy crawl mode requested")`

    Handle empty graph case: if ScanDirectory returns 0 docs, print "No markdown files found in <path>" to stderr and return nil (not an error — matches export behavior difference per CONTEXT.md where crawl is diagnostic).
