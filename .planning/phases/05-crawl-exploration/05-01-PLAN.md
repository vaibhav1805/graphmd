<feature>
  <name>Crawl Stats Engine</name>
  <files>internal/knowledge/crawl_stats.go, internal/knowledge/crawl_stats_test.go</files>
  <behavior>
    ComputeCrawlStats(*Graph) -> CrawlStats

    CrawlStats contains:
    - ComponentCount (int): len(graph.Nodes)
    - RelationshipCount (int): len(graph.Edges)
    - QualityScore (float64): weighted average of all edge confidence scores * 100 (percentage). 0 if no edges.
    - ComponentsByType (map[ComponentType][]string): node IDs grouped by ComponentType, each slice sorted alphabetically
    - ConfidenceDistribution ([]TierStats): one entry per non-zero tier, ordered by descending confidence (explicit first)
    - QualityWarnings ([]QualityWarning): detected graph quality issues

    TierStats struct:
    - Tier (ConfidenceTier)
    - RangeLow (float64): lower bound of tier (e.g., 0.95 for explicit)
    - RangeHigh (float64): upper bound of tier (1.0 for explicit, lower bound of next-higher tier for others)
    - Count (int)
    - Percentage (float64): count/total * 100

    Tier ranges derived from ScoreToTier boundaries:
    - explicit: [0.95, 1.0]
    - strong-inference: [0.75, 0.95)
    - moderate: [0.55, 0.75)
    - weak: [0.45, 0.55)
    - semantic: [0.42, 0.45)
    - threshold: [0.4, 0.42)

    QualityWarning struct:
    - Type (string): "orphan_node", "dangling_edge", "weak_only"
    - Message (string): human-readable description with count
    - Items ([]string): affected node/edge IDs, sorted

    Quality warning detection:
    - orphan_node: nodes with zero incoming AND zero outgoing edges (check BySource and ByTarget)
    - dangling_edge: edges where Source or Target is not in graph.Nodes
    - weak_only: nodes where ALL connected edges (both incoming and outgoing) have confidence < 0.55 (below moderate tier)

    Test cases:
    - Empty graph -> CrawlStats{0, 0, 0, empty map, nil, nil}
    - Graph with 3 nodes, 2 edges (confidence 0.8, 0.6) -> correct counts, QualityScore = (0.8+0.6)/2*100 = 70.0, distribution has strong-inference:1 and moderate:1
    - Graph with orphan node -> QualityWarning{Type:"orphan_node"} with the orphan ID
    - Graph with weak-only component (all edges < 0.55) -> QualityWarning{Type:"weak_only"}
    - Graph with edge referencing non-existent node -> QualityWarning{Type:"dangling_edge"}
    - All 6 tiers populated -> 6 TierStats entries in correct order
    - Single tier populated -> only 1 TierStats entry (others skipped)
    - Components grouped correctly by type with alphabetical sorting within each group
  </behavior>
  <implementation>
    Create `crawl_stats.go` in package knowledge:

    1. Define CrawlStats, TierStats, QualityWarning structs with json tags for later JSON formatting.

    2. Define tierBounds as a package-level var or func that maps ConfidenceTier -> (low, high) pairs derived from the ScoreToTier thresholds in confidence.go. Do NOT hardcode — derive from the same constants:
       - explicit: low=0.95, high=1.0
       - strong-inference: low=0.75, high=0.95
       - moderate: low=0.55, high=0.75
       - weak: low=0.45, high=0.55
       - semantic: low=0.42, high=0.45
       - threshold: low=0.4, high=0.42

    3. ComputeCrawlStats(*Graph) CrawlStats:
       - Count nodes and edges
       - Compute quality score (sum confidences / count * 100, guard div-by-zero)
       - Group components by type (iterate graph.Nodes, group by node.ComponentType, sort IDs within each group)
       - Compute confidence distribution (iterate graph.Edges, bucket by ScoreToTier, skip zero-count tiers, compute percentages)
       - Detect quality warnings (orphans, dangling, weak-only)
       - Return assembled CrawlStats

    4. Keep helper functions private: computeConfidenceDistribution, computeQualityScore, detectQualityWarnings, groupComponentsByType.
  </implementation>
</feature>
